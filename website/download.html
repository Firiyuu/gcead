Microsoft Windows

NOTE: The software has only been tested on 32-bit Windows XP systems.

If other Syntech software is on your computer:

  1. Download and run the installer: GcEad-1.2.1-windows-installer-20101016.exe
  2. Note that you will not be able to install the IDAC drivers for GcEad/2010, because they conflict with older drivers that were installed with previous Syntech software.
  3. In order to let GcEad/2010 work with the old drivers, you can AT YOUR OWN RISK install this software: libusb-win32-filter-bin-0.1.12.2.exe. However, it is only recommened for users running the 32-bit version of Windows XP, as with other versions it may cause very serious USB errors.
  4. By the end of November at latest, we'll try to have a version of GcEad/2010 available which is compatible with the old drivers.
If GcEad/2010 is the only Syntech software on your computer:
  1. Download and run the installer: GcEad-1.2.1-windows-installer-20101016.exe
  2. If you want to use the IDAC-2 or IDAC-4 acquisition hardware with GcEad/2010, then please choose to install the USB driver during installation.

Mac OS X

NOTE: The software has only been tested on version 10.6 of OS X. Also, IDAC-4 acquisition devices do not work on the Mac version of this software.
  1. Download GcEad-1.2.1-mac-20101016.dmg
  2. You can now run GC-EAD by double-clicking on the .dmg file

Linux

NOTE: The compiled linux version was only tested on Kubuntu linux.
  1. Download GcEad-1.2.1-linux-installer-20101016.bin
  2. Make the .bin file executable (e.g. from the command-line, run "chmod a+x GcEad-1.2.1-linux-installer-20101016.bin")
  3. Run the .bin file (e.g. from the command-line, run "./GcEad-1.2.1-linux-installer-20101016.bin")
  4. Icons to launch GC-EAD should now be available on your desktop and from the programs menu under "Science".

Source Code

The source code can be downloaded from the sourceforge SVN server using this url:
https://gcead.svn.sourceforge.net/svnroot/gcead/trunk

If you are using the command-line svn utility, the following command line should work:
svn co https://gcead.svn.sourceforge.net/svnroot/gcead/trunk gcead
